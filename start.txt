1. Создаем дерикторию проекта
2. Создаем вертуальное окружение и активируем
3. Создаем файл requirements.txt , куда указываем используемые библиотеки
4. Устонавлеваем библиотеки при помощьи pip3 install -r requirements.txt
5. Открываемм настройки : File -> Settings -> Project -> Python Interpreter и выбераете версию Python из окружение
6. Проверить что виртуальное окружение активирована в тирменале PyCharm
7. Создаем Django-проект , создается он с помощью : "django-admin startproject название_проекта ."
8. Создаем нужные приложение через : "python3 manage.py startapp название_приложение"

git init - инициализирует локальный репозиторий в текущий папке
git status - проверка текущего состояние репозитория
git add - добавление файлов в комит
git add . - добавление всех файлов в комит
git add - добавление конкретных файлов
changes to be committed - файлы , которые будут добавлены в версию
untracked changes - файлы , которые остаются вне версии
git commit - создание коммита (версии)
git commit -m 'Название коммита'
git push - отправка локальных коммитов на удаленный репозиторий
git push название_репозитория название_ветки

git remote - работа с удаленным репозиториями
git remote add - привязка удаленого репозитория
git remote add название ссылка_на_ репозиторий
got remote -v - срисок привязонных удаленных репозиториев
git remote rm название # исключение удаленного репозитория

git branch - работа с ветками
git branch - просмотр всех веток (активная ветка помичается * )
git branch название_ветки - создание новой ветки
git branch -D название_ветки - удаление ветки
git checkout название_ветки - переход на ветку
git checkout -b название_ветки - создание ветки и переход на неё

git pull - получение изменений из удаленого репозитория
git pull название_репозитория название ветки


git stash - прячет изменение
git stash apply - возвращает спрятанные изменение
